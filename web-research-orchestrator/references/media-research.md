# Multimedia Research

Use video and audio sources when they provide direct statements, demonstrations, interviews, or context unavailable in text. Do not let multimedia replace verifiable textual evidence when the claim is factual.

## YouTube

Use this order:

1. Capture video title, channel, publication date, description, and URL.
2. Check whether the channel is official or otherwise relevant.
3. Look for official transcripts, captions, description links, slides, blog posts, papers, or related announcements.
4. Use captions only when available and allowed.
5. For claims from a video, record the timestamp and cross-check with a textual source when possible.

Important limitation:

- YouTube Data API `captions.list` lists caption tracks.
- `captions.download` can retrieve caption text, but requires permission to edit the video. It is not a general public transcript API.

## Podcasts

Use this order:

1. Identify the podcast page and RSS feed.
2. Look for Podcasting 2.0 `podcast:transcript` tags in episode items.
3. If no transcript is available, use show notes, official pages, guest links, and related documents.
4. Record episode title, show, publisher, date, participants, URL, and timestamp or segment if available.

## Final report rules

- Do not make unsupported factual claims based only on audio/video.
- Label media-derived evidence clearly.
- Include timestamps when possible.
- Prefer "the speaker said" over "it is true that" unless independently verified.
